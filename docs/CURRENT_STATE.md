# DRIPLY — текущее состояние проекта

Обновлено: 23 июля 2026.

## Продукт

DRIPLY — мобильный fashion-маркетплейс для рынка СНГ. Основной сценарий: пользователь быстро просматривает товары в свайп-ленте, сохраняет интересное, открывает подробную карточку, пишет продавцу и может сам публиковать объявления.

Telegram-интеграция временно отложена. Сначала создаётся полноценный веб-продукт.

## Инфраструктура

- Frontend hosting: Vercel

## Региональная логика

Целевой рынок — СНГ.

Поддерживаемые страны и валюты:

- Россия — RUB / ₽
- Беларусь — BYN / Br
- Казахстан — KZT / ₸
- Украина — UAH / ₴
- Армения — AMD / ֏
- Грузия — GEL / ₾
- Кыргызстан — KGS / сом
- Узбекистан — UZS / сум
- Молдова — MDL / L

План определения региона:

1. IP-геолокация.
2. Запасной вариант: язык и часовой пояс устройства.
3. Ручной выбор страны пользователем имеет наивысший приоритет.
4. Выбранный регион сохраняется локально и позже в профиле пользователя.

## Важные UX-решения

- Главная лента не прокручивается вертикально.
- Остальные экраны прокручиваются вертикально.
- Фото переключаются тапом, а не горизонтальным свайпом, чтобы не конфликтовать со свайпом товара.
- Оригинальное фото хранится без обрезки.
- В ленте используется подготовленная обложка.
- В подробной карточке показывается полное изображение.
- До 10 фотографий на одно объявление.
- Ручной выбор региона обязателен, даже если работает автоопределение.

## Сейчас данные тестовые

Интерфейс пока использует моковые товары внутри frontend. Реальные CRUD-операции, пользователи, избранное, свайпы, сообщения и загрузка фото ещё не подключены к backend и Supabase.
